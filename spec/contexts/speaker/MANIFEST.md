# Speaker — Context Manifest

**Audience:** event organizers, conference programming committees, and their AI agents (SIDE).
**Context ID:** `hcp-speaker`
**Files:** 3. Optional add-on.

The Speaker context answers the questions an organizer's agent actually asks: *what does this person talk about, how do they deliver, and where have they spoken before?*

## File Set

| File | Purpose |
|------|---------|
| `SPEAKER_TOPICS.md` | Primary topics, topic combinations, emerging topics |
| `SPEAKER_STYLE.md` | Format preferences, audience interaction, practical requirements |
| `SPEAKER_APPEARANCES.md` | Appearance log with summary stats |

## Why It Exists in the Spec

Speaker is the **proof of extensibility**: a complete context added as a self-contained branch — own manifest, own templates, own interview flow in the hosted product — with zero changes to Human Core or HCP-pro. New contexts (entrepreneur, sport, research) follow this exact pattern.
